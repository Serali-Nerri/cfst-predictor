# 参数重命名与知识增强优化方案

## 1. 目标

基于 Lou et al. (2025) 的 KeML 思路，并结合当前仓库已经落地的 `Npl` 锚点建模主线，对现有方案做两类优化：

1. **命名优化**：统一公式符号、图表符号、CSV 字段名与代码内部命名。
2. **知识增强优化**：将当前“仅使用 `Npl` 锚点 + 单一 XGBoost”的做法，升级为 **baseline + 线性残差分支 + 非线性残差分支** 的 KeML 风格结构。

当前仓库已经具备的基础包括：

- 以 `Nexp (kN)` 为报告目标
- 以 `Npl (kN)` 为知识锚点
- 以 `r = (Nexp - Npl) / Npl` 为当前默认训练目标
- 以 `eta_u = Nexp / Npl` 作为归一化比值符号
- 在预测阶段将模型输出恢复回 `Nexp` 空间

这些能力说明当前实现已经具备 KeML 的 baseline 基础，但尚未显式拆分出论文中的线性分支与非线性分支。

---

## 2. Lou 2025 中相关符号的对应关系

### 2.1 `Npl` 对应的论文符号

当前仓库中的 `Npl` 对应 Lou 2025 中的：

\[
N_{pl}
\]

其物理含义为复合截面的塑性承载力：

\[
N_{pl} = A_c f_c + A_s f_y
\]

在 KeML 总框架中，Lou 2025 将经验式或知识基线统一记作：

\[
y_e
\]

因此当本项目采用 `Npl` 作为基线时，可写成：

\[
y_e = N_{pl}
\]

也就是说：

- **物理量符号**：`N_{pl}`
- **KeML 中的角色名**：`y_e`
- **当前工程中的具体实现**：`Npl (kN)`

### 2.2 `eta_u` 的角色

当前统一定义：

\[
\eta_u = \frac{N_{exp}}{N_{pl}}
\]

它表示实验承载力相对于 `Npl` 基线的无量纲比值。

相较于旧的 `psi` 记号，`eta_u` 的优点是：

1. 更贴近论文图表中对归一化物理量的表达习惯。
2. 不再引入与旧实现耦合的历史符号负担。
3. 更容易与后续残差目标 `r` 形成清晰关系。

---

## 3. 推荐命名方案

### 3.1 总原则

建议以后区分三层命名：

1. **公式符号**：用于论文、图表、报告
2. **代码内部名**：用于 Python 变量、函数、metadata
3. **CSV/展示字段名**：用于数据表、训练输入、导出结果

目标是让三者语义一致，但不要求完全同名。

### 3.2 当前推荐映射

| 用途 | 公式符号 | 代码名 / CSV名 | 说明 |
|---|---|---|---|
| 塑性承载基线 | `N_{pl}` | `Npl (kN)` | 对应 Lou 2025 的塑性承载力 |
| 归一化承载力比 | `\eta_u` | `eta_u` | 表示 `Nexp / Npl` |
| 相对残差 | `r` | `r` | 表示 `(Nexp - Npl) / Npl` |

### 3.3 推荐结论

- **保留 `Npl (kN)` 作为当前工程列名**
- **统一使用 `eta_u` 替代旧的 `psi`**
- **统一使用 `r` 作为训练主目标**
- **不再使用 `strength_ratio` / `relative_residual` 这类语义化中间名**

---

## 4. 当前模型与 Lou 2025 的差距

### 4.1 当前实现的本质

当前仓库已经实现的是：

\[
\hat N = N_{pl} (1 + \hat r)
\]

其中：

\[
\hat r = \hat r_L + \hat r_N
\]

也就是说，当前主线已经从单纯的归一化目标建模，推进到了更接近 KeML 的残差建模形式。

### 4.2 尚未完全实现的部分

Lou 2025 的关键增强点是把修正项拆成：

\[
\delta(x, y_e) = f_L(x, y_e) + f_N(x, y_e)
\]

若本项目令：

\[
y_e = N_{pl}
\]

则当前目标形式可写为：

\[
\hat N = N_{pl} (1 + \hat r_L + \hat r_N)
\]

也就是：

- `r_L`：线性残差分支
- `r_N`：非线性残差分支

当前仓库已经接入这一路线，但后续仍需要继续做实验验证与消融对比。

---

## 5. 推荐的知识增强优化框架

### 5.1 总体结构

推荐并已开始落地的训练主线为：

\[
\hat N = N_{pl}(1 + \hat r)
\]

其中：

\[
\hat r = \hat r_L + \hat r_N
\]

- `Npl` 提供主尺度与知识锚点
- `r_L` 学习系统性线性偏差
- `r_N` 学习高阶非线性耦合

### 5.2 训练目标

当前推荐训练目标为：

\[
r = \frac{N_{exp} - N_{pl}}{N_{pl}}
\]

它与 `eta_u` 的关系是：

\[
r = \eta_u - 1
\]

`r` 的工程含义清楚：

- `r = 0`：说明 baseline 已准确
- `r > 0`：实验值高于 baseline
- `r < 0`：实验值低于 baseline

这更适合做线性残差与非线性残差分解。

### 5.3 关于长尾数据：为什么推荐 `r`，而不是继续主打 `log(psi)`

当前仓库早期使用 `log(psi)` 的直接动机，是希望缓解高承载力样本形成的长尾与尺度不均衡问题。这个思路在统计上是合理的，但从当前论文叙事和知识增强建模的角度，更推荐用：

\[
r = \frac{N_{exp} - N_{pl}}{N_{pl}}
\]

原因如下：

#### A. 与 Lou 2025 的叙事更一致

`r` 本质上就是“相对知识基线的残差”，与 KeML 中：

\[
y_f = y_e + \delta(x, y_e)
\]

天然一致。

#### B. 仍然具备缓解长尾的效果

目标从原始 `Nexp` 变为相对基线残差 `r` 后，模型不再直接承担全部强度尺度，而是只学习相对于 `Npl` 的修正量。对高承载力样本来说，这本身就能缩小目标尺度跨度。

#### C. 更适合解释与作图

`r = 0` 表示 baseline 准确；
`r > 0` 表示实验值高于 baseline；
`r < 0` 表示实验值低于 baseline。

因此，当前更适合把 `r` 作为主叙事，而把旧的 `log(psi)` 路线保留为历史对照。

---

## 6. 线性残差分支：Ridge 学 `r_L`

### 6.1 含义

这里的 “Ridge 学 `r_L`” 指的是：

- 先定义目标 `r`
- 再用一个带 L2 正则化的线性回归模型去拟合 `r` 中可由低维机理特征解释的线性部分

写成公式：

\[
\hat r_L = w^T z_L + b
\]

其中 `z_L` 是一组无量纲、低维、可解释的机理特征。

### 6.2 为什么推荐 Ridge

Lou 2025 在线性分支中使用的是 **single neuron with no activation**，本质上就是一个线性层。

在本项目中优先推荐 Ridge，而不是普通线性回归，原因是：

1. 当前特征之间相关性较强
2. 数据量较小，直接 OLS 更容易不稳定
3. Ridge 保持线性可解释性，同时更稳健

因此，Ridge 可以看作 Lou 2025 线性分支在本项目中的工程化实现。

### 6.3 推荐输入特征 `z_L`

建议线性分支优先使用以下解释性强的无量纲特征：

- `e/h`
- `lambda_bar`
- `b/h`
- `r0/h`
- `b/t` 或 `h/t`
- `ke`
- `xi`
- `L/h`
- `R (%)`

这些特征更适合表达：

- 偏压程度
- 稳定效应
- 截面形状
- 约束效应
- 再生骨料替代率

### 6.4 线性分支输出解释

`r_L` 的作用不是替代全部模型，而是捕捉那些方向明确、解释性强、跨样本分布较稳定的系统偏差。

---

## 7. 非线性残差分支：XGBoost 学 `r_N`

### 7.1 目标定义

在线性分支训练完成后，定义剩余残差：

\[
r^{(1)} = r - \hat r_L
\]

然后让 XGBoost 拟合：

\[
\hat r_N \approx r^{(1)}
\]

最终：

\[
\hat r = \hat r_L + \hat r_N
\]

### 7.2 为什么这样拆分

相比原先“单个 XGBoost 吃掉全部关系”的做法，这种方式有三个优势：

1. **更接近 Lou 2025 的 KeML 结构**
2. **更可解释**：可以单独看线性分支学到了什么
3. **更稳健**：复杂模型只需学习剩余非线性部分，优化难度更低

### 7.3 非线性分支推荐特征

非线性分支可以继续使用当前主线中的完整机理特征集合，但建议进行分层管理：

#### A. baseline 层

- `Npl` 只作为锚点，不作为默认普通强特征重复输入

#### B. 线性知识层

- `e/h`
- `lambda_bar`
- `b/h`
- `r0/h`
- `ke`
- `xi`
- `L/h`
- `R (%)`

#### C. 非线性层

- `Ac`
- `As`
- `sigma_re`
- `section_family`
- `axial_flag`
- 偏压相关派生量
- 其他交互敏感特征

---

## 8. 关于 `Npl` 是否继续作为普通特征输入

### 8.1 不建议默认重复输入 raw `Npl`

当前若采用：

\[
\hat N = N_{pl}(1 + \hat r)
\]

则 `Npl` 已经作为 baseline 注入一次。

这时若又把 raw `Npl` 作为普通强特征输入给 XGBoost，容易造成：

- 冗余信息重复注入
- 模型过度依赖强尺度特征
- 削弱残差学习的清晰性

### 8.2 更推荐保留的信息

推荐保留这些更有增量信息的特征：

- `Ac * fc`
- `As * fy`
- `lambda_bar`
- `ke`
- `xi`
- 偏压与稳定相关量

### 8.3 建议策略

- 默认：**不把 raw `Npl` 作为普通特征强制输入**
- 消融：可以做一组“是否显式加入 `Npl` 特征”的附加实验

---

## 9. 推荐训练流程

### Step A：构造机理量

对所有样本计算：

- `Ac`
- `As`
- `ke`
- `xi`
- `lambda_bar`
- `e/h`
- `b/h`
- `L/h`
- `Npl`
- `eta_u = Nexp / Npl`
- `r = (Nexp - Npl) / Npl`

### Step B：确定数据切分

继续保留当前主线：

- train/test split
- regression stratified
- 在 train 上做 CV
- 在 `Nexp` 空间汇报最终指标

### Step C：训练线性分支

用 Ridge 拟合：

\[
r \approx r_L
\]

### Step D：训练非线性分支

构造：

\[
r^{(1)} = r - \hat r_L
\]

再用 XGBoost 拟合：

\[
r^{(1)} \approx r_N
\]

### Step E：合成输出

\[
\hat r = \hat r_L + \hat r_N
\]

\[
\hat N = N_{pl}(1 + \hat r)
\]

### Step F：评估

建议保留并强调以下三组比较：

1. `Npl` baseline
2. `Npl + Ridge` 线性修正
3. `Npl + Ridge + XGBoost` 完整 KeML 方案

这样可以直接量化每一层知识嵌入带来的收益。

## 10. 建议的代码重构方向

### 10.1 `scripts/compute_feature_parameters.py`

建议：

- 保留：`Npl (kN)`
- 保留：`eta_u`
- 保留：`r`
- 不再新增语义化中间名

### 10.2 `src/domain_features.py`

建议逐步重构为：

- 保留对 `Npl` 的恢复逻辑
- 保留 `eta_u` / `r` 的统一命名支持
- 将目标模式统一为：
  - `eta_u_over_npl`
  - `r_over_npl`
- 同时允许旧模式作为兼容别名读取，但不再作为主叙事暴露

### 10.3 `src/data_loader.py`

建议：

- 训练时优先读取 `r`
- 兼容旧版 `psi` / 旧语义化列的读取映射，仅用于平滑迁移
- 明确排除不应重复注入的目标派生列与 baseline 派生列

### 10.4 `train.py`

建议保持两阶段训练逻辑：

1. 训练 Ridge 线性分支
2. 训练 XGBoost 非线性分支
3. 保存组合模型 metadata
4. 将 KeML 相关配置纳入训练上下文与 Optuna fingerprint

### 10.5 `src/predictor.py`

建议支持如下组合预测逻辑：

1. 先算 `r_L`
2. 再算 `r_N`
3. 合成 `r`
4. 根据 `Npl` 恢复 `Nexp`

## 11. 推荐迁移策略

### Phase 1：命名迁移

先完成：

- 图表符号：`psi -> eta_u`
- CSV 列：统一为 `eta_u` 与 `r`
- 保留旧列读取兼容，但不再继续扩散旧命名

### Phase 2：训练目标切换

将主训练目标统一到：

\[
r
\]

目标：正式切换为“相对基线残差”主线。

### Phase 3：引入 KeML 双分支

在训练流程中增加：

- Ridge 线性分支
- XGBoost 非线性分支
- 组合输出

目标：完成与 Lou 2025 更一致的知识增强结构升级。

---

## 12. 最终推荐结论

### 12.1 命名

- `Npl` 对应 Lou 2025 中的 `N_{pl}`
- 在 KeML 中其角色可写为 `y_e = N_{pl}`
- 归一化强度比统一写为：

\[
\eta_u = \frac{N_{exp}}{N_{pl}}
\]

- 训练目标统一写为：

\[
r = \frac{N_{exp} - N_{pl}}{N_{pl}}
\]

### 12.2 模型结构

推荐将当前主线从：

- `Npl` baseline + 单一 XGBoost

升级为：

- `Npl` baseline + Ridge 线性残差 + XGBoost 非线性残差

即：

\[
\hat N = N_{pl}(1 + \hat r_L + \hat r_N)
\]

### 12.3 工程策略

最稳妥的落地顺序是：

1. 先完成命名统一到 `eta_u / r`
2. 再切换训练目标到 `r`
3. 最后引入 `Ridge + XGBoost` 的双分支 KeML 结构

这样可以在保持当前主线可运行的前提下，逐步升级到更接近 Lou 2025 的知识增强框架。
