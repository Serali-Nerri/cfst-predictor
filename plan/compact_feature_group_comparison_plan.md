# 多截面 CFST 柱统一预测框架：紧凑特征组对比实验计划

## 1. 目的

在“多截面 CFST 柱极限承载力统一预测框架”背景下，围绕统一截面、形状效应、整体稳定与偏心受压机理，筛选出参数个数尽可能少、但物理机理覆盖完整的紧凑特征组。

本轮计划不再优先追求原始几何变量堆叠，而是优先比较“等效统一参数 + 稳定参数 + 偏心参数”的组合效果。

## 2. 实验统一要求

所有候选特征组均执行同一套训练与评估流程：

1. 报告目标固定为 `Nexp (kN)`
2. 训练主线保持当前默认主线：`eta_u_over_npl + log + KeML`
3. 每一组特征参数 **都要单独运行 100 轮超参数调优（Optuna）**
4. 每组实验均保留独立输出目录、独立 best params、独立 study
5. 统一汇报：
   - Train: `RMSE / MAE / R² / MAPE / μ / COV / a20-index`
   - Test: `RMSE / MAE / R² / MAPE / μ / COV / a20-index`
   - CV: `J / RMSE / MAE / R² / MAPE / μ / COV / a20-index`
6. 统一比较 regime analysis，重点关注：
   - `section_family`
   - `slenderness_state`
   - `eccentricity_severity`
   - `confinement_level`

## 3. 候选特征组

### Group A：统一机理紧凑主线组（8个）

**特征参数：**
- `fy (MPa)`
- `fc (MPa)`
- `Re (mm)`
- `te (mm)`
- `ke`
- `lambda_bar`
- `e/h`
- `e_bar`

**选取理由：**
- `fy + fc` 提供钢管与混凝土强度基础
- `Re + te` 对应等效圆柱统一表达
- `ke` 直接体现不同截面形状下的有效约束差异
- `lambda_bar` 统一反映整体稳定与材料水平的耦合
- `e/h + e_bar` 共同描述偏心幅值与归一化弯曲效应

**定位：**
- 推荐作为后续主线候选
- 参数少、机理闭环完整、跨截面统一性强

---

### Group B：偏心增强组（9个）

**特征参数：**
- `fy (MPa)`
- `fc (MPa)`
- `Re (mm)`
- `te (mm)`
- `ke`
- `lambda_bar`
- `e/h`
- `e_bar`
- `e1/e2`

**选取理由：**
- 在 Group A 基础上保留 `e1/e2`
- 用于区分单曲率与双曲率，以及不同偏心端部模式
- 适合检验偏心模式信息是否能显著改善泛化表现

**风险提醒：**
- `e1/e2` 在小偏心或分母接近 0 时可能数值不稳定
- 更适合作为偏心专题增强组，而非默认主线

---

### Group C：形状解释增强组（9个）

**特征参数：**
- `fy (MPa)`
- `fc (MPa)`
- `Re (mm)`
- `te (mm)`
- `ke`
- `lambda_bar`
- `r0/h`
- `b/h`
- `e_bar`

**选取理由：**
- 在统一等效参数外，显式加入两项形状控制量
- `r0/h` 描述角径比与截面类型连续演化
- `b/h` 描述截面扁平化程度
- 配合 `ke` 可增强模型对多截面形状效应的可解释性

**定位：**
- 适合做“统一截面框架”的论文解释组
- 适合后续做 SHAP / 特征贡献分析

---

### Group D：超紧凑组（7个）

**特征参数：**
- `fy (MPa)`
- `fc (MPa)`
- `Re (mm)`
- `te (mm)`
- `ke`
- `lambda_bar`
- `e_bar`

**选取理由：**
- 在尽量少的参数下保留材料、统一截面、稳定、偏心四类核心机理
- 用于检验极简方案下是否仍可维持较高预测性能

**定位：**
- 作为下限基线组
- 用于测试是否存在进一步压缩特征维数的空间

## 4. 对比重点

本轮对比不只看 `RMSE / R²`，还要重点看：

1. `μ / COV / a20-index`
   - 这是论文/工程口径下更关键的评价指标
2. `Test` 与 `CV` 的一致性
   - 判断是否存在对某组特征的过拟合依赖
3. 不同 regime 的误差分布
   - 判断某组是否只在某一截面族或某一偏心区间表现更好
4. 偏心相关特征的收益
   - 核心看 Group B 相对 Group A 是否稳定提升
5. 形状显式参数的收益
   - 核心看 Group C 相对 Group A 是否带来更好的跨截面泛化
6. 超紧凑组的性能损失
   - 核心看 Group D 相对 Group A 的性能折损是否可接受

## 5. 推荐实验顺序

建议按以下顺序推进：

1. Group A：统一机理紧凑主线组
2. Group B：偏心增强组
3. Group C：形状解释增强组
4. Group D：超紧凑组

这样可以先得到最稳妥的主线候选，再逐步检验偏心增强、形状增强和进一步压缩维度的收益。

## 6. 每组实验的执行要求

对于每一组候选特征组，都必须满足：

- 单独配置 `columns_to_drop`
- 单独设置输出目录
- 单独设置 `best_params_path`
- 单独设置 Optuna study 名称/上下文
- **单独执行 100 轮 Optuna 调优**
- 调优完成后再汇总 train/test/CV 指标
- 保留 `evaluation_report.json`、`training_metadata.json` 与图表输出

## 7. 最终交付形式

本轮对比实验完成后，最终应形成一张汇总表，至少包含：

- 特征组名称
- 参数数量
- 特征列表
- Train 指标
- Test 指标
- CV 指标
- 主要 regime 短板
- 综合建议排序

## 8. 当前建议结论（实验前）

在尚未完成 100 轮调优对比之前，当前优先级建议如下：

1. **Group A**：最适合做统一预测主线
2. **Group C**：最适合做形状统一解释
3. **Group B**：最适合做偏心增强验证
4. **Group D**：最适合做极简下限基线
