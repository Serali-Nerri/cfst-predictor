# 下一阶段实验计划：CFST 统一预测框架的 ML 导向优化路线

## 1. 背景与当前判断

当前仓库已经完成并验证过的优化包括：

- 紧凑特征组筛选与对比（A / B / C / D）
- `compact_group_B` 基础上的迭代优化
  - 分层切分升级
  - 样本加权
  - 偏心变量精简
- `compact_group_B` 的特征重构筛选
  - `B4 = B + b/h`
  - `B6 = B + b/h + r0/h`
- Box-Cox 目标族的初步扫描（已完成 B4 的部分结果）

当前阶段的核心结论是：

1. **仅靠继续增加 Optuna 轮数，不足以解决 `R²` 与 `COV` 的结构性拉扯**
2. **`b/h` 是当前最有价值的新增形状变量**
3. **`e1/e2` 仍然不建议删除**
4. **B4 与 B6 是当前最值得继续拓展的两条紧凑特征路线**
5. **`log` 与 `r` 很可能都只是代理目标（proxy）**，并未真正从训练损失层面对齐最终关心的 `R² / COV / μ / a20-index`

因此，下一阶段优化应从“继续微调特征”转向“更贴近机器学习本体”的方向：

- 损失函数/概率假设优化
- 模型家族扩展
- 不确定性/分布式回归
- 多任务/多头学习
- 集成与校准

---

## 2. 下一阶段总目标

围绕以下目标继续优化：

- Test `R²` 尽可能贴近或超过 `0.99`
- 同时尽量将 `COV` 压低至 `<= 0.10`
- 兼顾 `μ` 接近 1 与 `a20-index` 提升

这一阶段不再只把注意力放在土木机理解释上，而是将问题明确视为一个：

> **正值、偏态、异方差、表格数据回归问题**

来引入更合适的机器学习建模方式。

同时需要明确：

- 当前阶段仍属于 **协议探索阶段**
- 本计划的任务是继续完成关键实验，帮助后续冻结：
  - 最终共享评估协议
  - 最终用于多模型公平比较的特征组
  - 最终 target transform / training target 设定
- 因此，本计划现阶段**不提前冻结**共享评估协议与最终特征组，而是在实验结束后根据证据统一定稿

---

## 4. 已验证但暂不优先继续的方向

以下方向已经做过探索，当前不作为下一阶段最高优先级：

### 4.1 继续细调分层切分

已验证：

- `lambda_bar + e/h` 辅助分层
- `lambda_bar + e_bar` 辅助分层

结果：

- 额外偏心辅助分层在当前稳定约束下没有形成有效附加分层
- 对最终指标没有带来实质改善

### 4.2 继续仅靠样本加权扫描

已验证：

- `e/h > 0.1` 权重 1.5
- `e/h > 0.1` 权重 2.0
- `e/h > 0.2` 权重 2.0

结果：

- 样本加权可以带来局部改善
- 但仍不足以把 `R²` 与 `COV` 同时推到目标区间
- 更像是次级调节手段，而不是根本突破点

### 4.3 只在 `log / r / Box-Cox` 上做更细扫描

结果：

- 仅靠 target transform 本身，目前还没有显示出足够强的“同时改善 `R²` 与 `COV`”能力
- 它仍然值得保留，但不应再作为唯一主线

## 5. 下一阶段的核心优化方向

## 5.1 方向一：继续完成 B4 / B6 的 Box-Cox 扫描

### 目标

在 B4 与 B6 两组上继续验证：

- `log`
- `boxcox_0.25`
- `boxcox_0.50`
- `boxcox_0.75`
- `boxcox_1.00`

是否存在优于 `log` 的折中点。

### 当前中间结论

B4 已完成的部分结果显示：

- `log` 仍然是当前最均衡点
- 中间 Box-Cox λ 值更容易把模型推向更高 `R²`，但也会明显恶化 `COV`

### 下一步建议

- 先补完 `B4_boxcox_1.00`
- 再补完 B6 五组
- 形成一张完整的 `λ-R²-COV` 曲线

### 价值

- 明确 transform 是否仍然值得继续投入
- 为后续“损失函数层优化”提供对照基线

补充说明：

- 当前继续完成 Box-Cox 扫描，是为了补齐协议探索阶段的证据链
- 这并不意味着 Box-Cox 已被判断为长期主线
- 它在本计划中的角色更接近：帮助确认后续共享 target 设定是否仍值得继续围绕 transform 本身展开

---

## 5.2 方向二：补“反变换偏差修正”，优先验证 log 路线

### 核心思想

当前 `log` 路线在原始空间反变换时，存在经典的 bias 问题：

- `E[exp(X)] != exp(E[X])`

可考虑引入：

- **Duan smearing estimator**

### 目标

不改变当前模型结构，只在 `log` 路线上增加反变换偏差修正，观察是否改善：

- `μ`
- `COV`
- 原始空间下的误差稳定性

### 建议优先组

- B4 + log + smearing
- B6 + log + smearing

### 价值

- 改动成本低
- 可能直接改善工程口径指标
- 是最值得优先补的小成本实验之一

---

## 5.3 方向三：引入正值偏态目标的概率建模

这是下一阶段最值得重点投入的 ML 主线。

### 3.1 Gamma / Tweedie deviance

#### 思路

将回归问题视为：

- 正值目标
- 右偏分布
- 方差随均值变化

而不是普通高斯 + MSE 回归。

#### 目标

尝试更适合正值偏态目标的训练目标，替代当前的 `reg:squarederror` 逻辑。

#### 推荐实验顺序

1. 先在 B4 上试
2. 再在 B6 上试
3. 与 `log` 路线直接对比

#### 价值

- 更贴近当前目标分布特性
- 有机会缓解 `R²` 与 `COV` 的结构性冲突

---

### 3.2 Lognormal likelihood

#### 思路

不只预测一个点，而是假设：

- `log(y)` 服从正态分布
- 模型学习该分布的参数

#### 目标

将当前 “log + 普通回归” 升级为：

- 具有概率解释的正值回归
- 更自然地兼顾比例误差结构

#### 建议优先组

- B4 + lognormal
- B6 + lognormal

#### 价值

- 与当前 log 路线最接近
- 迁移成本相对较低
- 更有理论基础去改善 `μ / COV`

---

### 3.3 异方差回归（同时预测均值和方差）

#### 思路

允许模型输出：

- 均值 `μ(x)`
- 方差 `σ(x)`

而不是假设所有样本噪声水平一致。

#### 目标

显式建模不同工况下的误差不稳定性。

#### 适用场景

- 偏心程度不同
- 截面族不同
- 细长比区间不同

这些 regime 可能天然具有不同方差。

#### 价值

- 更自然地处理异方差问题
- 有助于改善 ratio-based metric 的稳定性

---

### 3.4 Quantile / distributional regression

#### 思路

不只预测点值，而是预测：

- 分位点（如 0.1 / 0.5 / 0.9）
- 或整个条件分布的参数

#### 目标

理解并利用不同样本的不确定性分布结构，而不只是追求一个平均点预测。

#### 价值

- 有助于理解“哪些样本/哪些 regime 导致 COV 恶化”
- 为后续更稳健的点预测/集成提供依据

---

## 5.4 方向四：在统一框架下开展多模型 backbone 公平比较

### 核心定位

下一阶段的多模型实验，不应直接与 KeML、Box-Cox、样本加权、目标族扫描等全部变量混在同一轮解释，而应明确区分：

1. **backbone comparison（模型骨干比较）**
2. **framework enhancement comparison（框架增强比较）**

前者回答的问题是：

> 在统一多截面输入框架下，哪个基础模型最强？

后者回答的问题是：

> 在最优 backbone 的基础上，哪些增强模块最有效？

### 当前拟定的第一轮模型池

当前建议纳入第一轮公平比较的模型池为：

- XGBoost
- Random Forest (RF)
- CatBoost
- LightGBM
- BP 神经网络（ANN / MLP）

其中：

- `XGBoost / CatBoost / LightGBM` 为重点 backbone 候选
- `RF / BP` 作为传统树模型与经典神经网络对照组

### 当前阶段的比较策略

本计划现阶段暂不冻结共享协议，但在最终进入多模型公平比较前，建议采用如下思路：

- 共享数据集
- 共享 train/test split 与 CV 方案
- 共享评价指标（`R² / RMSE / MAE / μ / COV / a20-index`）
- 共享调参预算
- 每个模型在自己的搜索空间里**各自调优**，而不是使用完全相同的固定超参数

换言之：

> 公平比较并不意味着所有模型用同一组超参数，而是意味着所有模型在相同协议、相同预算下进行各自调优。

### 推荐执行顺序

1. 先完成本计划中的协议探索实验
2. 再确定最终用于 benchmark 的共享特征组和 target 路线
3. 然后开展五模型 backbone 公平比较
4. 最后只对胜出模型再做 KeML、distributional objective、异方差回归、集成等增强实验

## 5.5 方向五：扩展模型家族，而不只依赖 XGBoost

### 5.1 CatBoost

#### 推荐程度

**高优先级**

#### 原因

- 是当前最值得增加的强 tabular baseline
- 对表格数据与非线性交互常常很稳
- 很适合用于验证“当前瓶颈到底是目标函数还是模型家族”

#### 推荐实验顺序

- B4 + CatBoost
- B6 + CatBoost

先保持目标族与当前路线一致，再比较表现。

---

### 5.2 LightGBM

#### 推荐程度

**高优先级**

#### 原因

- 适合快速试不同 objective
- 对正值偏态目标的 objective 扩展更方便

#### 推荐实验顺序

- B4 + LightGBM + baseline objective
- B4 + LightGBM + Tweedie/Gamma-style objective
- 必要时再扩到 B6

---

### 5.3 深度学习表格模型

#### 推荐程度

**中后期再上**

推荐优先级：

1. ResNet-style MLP
2. FT-Transformer
3. SAINT / TabTransformer

#### 原因

- 当前任务仍然是典型的 engineered tabular regression
- 从文献与经验看，中小型 tabular 场景下树模型通常仍更强、更稳
- DL 更适合作为后续补充，而不是第一优先主线

---

## 5.5 方向五：多任务 / 多头学习

### 思路

用一个模型同时预测：

- 原始目标相关头（如 `eta_u`）
- `log(eta_u)` 头
- 不确定性/方差头
- 可选 quantile 头

### 目标

让训练过程同时看到：

- 绝对误差结构
- 相对误差结构
- 不确定性结构

### 价值

- 有机会同时改善 `R²` 与 `COV`
- 比单一 target transform 更灵活

### 推荐时间点

- 在单目标概率回归基线跑过之后再做
- 不建议作为第一批实验立即投入

---

## 5.6 方向六：模型集成与校准

### 思路

在强单模型基线上进一步尝试：

- XGBoost
- LightGBM
- CatBoost
- （后续可加 NN）

做简单平均、加权平均或 stacking。

### 目标

- 降低方差
- 改善原始空间预测稳定性
- 进一步压低 `COV`

### 价值

- 在表格回归任务里往往非常有效
- 也是后期冲最终上限的重要手段

---

## 6. 推荐实验优先级（按研究阶段排序）

### 阶段 A：协议探索阶段（当前优先完成）

1. **补完 B4 / B6 的 Box-Cox 扫描**
2. **给 B4 / B6 的 `log` 路线补 Duan smearing 修正**
3. **继续确认最适合进入统一 benchmark 的候选特征组与 target 路线**

### 阶段 B：backbone 公平比较阶段

在阶段 A 完成后，再冻结共享协议，并开展以下五模型公平比较：

1. **XGBoost**
2. **RF**
3. **CatBoost**
4. **LightGBM**
5. **BP**

这一阶段要求：

- 共享数据协议
- 共享特征组
- 共享 target 设定
- 共享评价指标
- 共享调参预算
- 各模型在自己的搜索空间里各自调优

### 阶段 C：优胜模型增强阶段

在阶段 B 筛出前 1~2 个 backbone 后，再优先尝试：

1. **Gamma / Tweedie deviance 类目标**
2. **lognormal likelihood / 异方差回归**
3. **Quantile / distributional regression**
4. **多任务 / 多头学习**
5. **集成 / stacking**
6. **FT-Transformer / ResNet-style MLP / SAINT**

## 8. 本计划的核心思想

当前仓库已经把：

- 特征组结构
- 分层切分
- KeML
- Optuna
- 选模目标平衡 `R² + COV`

这些方面都做到了较高完成度。

下一阶段最值得投入的，不再是继续小修小补，而是转向：

> **从训练损失、概率分布假设、模型家族和不确定性建模层面，重新定义“什么叫拟合得好”。**

这是最有可能打破当前 `R²` 与 `COV` 结构性拉扯的方向。
