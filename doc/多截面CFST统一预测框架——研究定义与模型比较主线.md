# 多截面 CFST 统一预测框架——研究定义与模型比较主线

## 1. 当前工作的定义

当前工作的核心定义为：

> **面向多截面统一输入框架的 CFST 柱极限承载力预测研究。**
>
> 在统一的多截面机理表达思路下，构建适用于不同截面形式 CFST 柱的机器学习预测框架，并在该框架内对多种模型进行系统比较，筛选出最优模型与后续最有潜力的增强路线。

这意味着，当前研究的重点已经不再只是“调优某一个单独模型”，而是：

1. 建立一个**多截面统一预测框架**；
2. 在该框架内完成**特征组筛选与结构优化**；
3. 在此基础上完成**多模型公平比较**；
4. 再对最优模型继续做知识增强、目标变换、损失函数优化等扩展。

---

## 2. 当前主题的主线框架

当前阶段的主题主线可概括为：

### 2.1 第一主线：多截面统一输入框架

核心目标是将不同截面形式的 CFST 柱放到统一的输入表达下，使模型不是针对单一截面做局部拟合，而是在统一机理空间中学习：

- 材料强度效应
- 统一截面等效与形状效应
- 整体稳定性效应
- 偏心受压效应

这一主线强调：

- `Re / te / ke` 等统一截面变量
- `lambda_bar` 等稳定性变量
- `e/h / e_bar / e1/e2` 等偏心变量
- `b/h / r0/h` 等形状解释变量

### 2.2 第二主线：在统一框架内进行多模型比较

在统一输入框架下，不直接预设某一模型就是最终最优，而是将多种代表性模型放入同一研究框架中比较，观察它们在：

- 拟合精度（`R² / RMSE / MAE`）
- 工程口径稳定性（`μ / COV / a20-index`）

上的表现差异。

### 2.3 第三主线：在最优模型上做进一步增强

在多模型比较完成后，再对入围模型进一步考虑：

- KeML 知识增强
- target transform / Box-Cox / log 路线
- 更贴近机器学习本体的损失函数与概率建模
- 可能的集成与不确定性建模

---

## 3. 当前拟定的模型池

当前阶段拟定的模型池为以下五类：

1. **XGBoost**
2. **Random Forest (RF)**
3. **CatBoost**
4. **LightGBM**
5. **BP 神经网络（ANN / MLP）**

这五类模型在当前研究中的定位如下。

### 3.1 XGBoost

- 当前仓库中最成熟、最深入开发的主力模型
- 也是当前统一框架、KeML、分层划分、target transform 等实验最主要的承载后端
- 后续仍然作为最核心的参考 backbone

### 3.2 RF

- 代表经典 bagging 树模型
- 作为非 boosting 树模型的传统基线
- 更侧重比较不同树模型机制下的表现差异

### 3.3 CatBoost

- 当前最值得纳入的强 tabular baseline 之一
- 用于比较更强的 boosting 树模型在当前统一框架下是否优于 XGBoost
- 在后续模型比较中属于重点竞争模型

### 3.4 LightGBM

- 作为另一类强 tabular boosting 模型纳入模型池
- 对后续尝试不同 objective（如偏态正值目标相关目标）也更方便
- 在当前研究中与 XGBoost、CatBoost 同属于重点 backbone 候选

### 3.5 BP 神经网络

- 作为经典神经网络基线模型
- 主要用于完成“树模型 vs 神经网络”的基础比较
- 在论文结构中有助于体现模型类型覆盖的完整性

## 4. 关于 KeML 的定位

当前应明确：

> **KeML 不是一个独立模型，而是一个知识增强框架模块。**

它的作用是在已有经验公式/机理知识的基础上，引入机器学习模块对残差或偏差进行修正，从而形成“经验知识 + 数据驱动”的混合框架。

因此，后续实验中要区分两层比较：

### 4.1 Backbone comparison（模型骨干比较）
比较：

- XGBoost
- RF
- CatBoost
- LightGBM
- BP

回答的问题是：

> 在统一特征框架下，哪一个基础模型最强？

### 4.2 Framework enhancement comparison（框架增强比较）
在筛选出的最优或前二模型上，再比较：

- 原始模型
- KeML 增强模型
- target transform / Box-Cox / log 路线
- 后续更高阶损失函数与概率建模路线

回答的问题是：

> 在已选 backbone 的基础上，哪些框架增强模块最有效？

## 5. 当前阶段暂不冻结的内容

为了避免过早锁定协议、影响后续结论解释，以下内容在当前阶段**暂不最终冻结**：

### 5.1 共享评估协议的最终版本

虽然最终一定需要统一：

- 数据切分方式
- CV 方式
- 调参预算
- 评价指标
- 随机种子

但当前阶段暂不做最终定稿。

原因：

- 需要先完成 `plan/next_stage_ml_optimization_plan.md` 中的关键实验
- 根据实验结果，再决定最终最合理的共享协议形式

### 5.2 最终用于多模型比较的特征组

当前已经筛出的高潜力候选包括：

- `B4`
- `B6`

但现阶段暂不最终锁定“多模型公平比较时只用哪一组”。

原因：

- 仍需等待下一阶段实验（包括 Box-Cox 补完、后续 ML 路线验证）进一步确认
- 特征组不应在证据尚未完整时过早冻结

### 5.3 最终 target transform 与训练目标设定

当前已知：

- `log` 在若干结果中仍然较均衡
- `r` 与部分 Box-Cox 中间点可能带来更高 `R²`
- 但是否真的适合作为最终共享协议的一部分，仍需更多实验验证

因此，target transform 暂不在当前文档中最终锁定。

尽管共享协议和最终特征组暂不冻结，但以下事项已经可以明确：

当前研究的主题不是单一模型调优，而是：

> **多截面统一输入框架下的 CFST 柱极限承载力机器学习预测研究，并在该框架中开展多模型比较。**

### 6.2 模型池已经初步确定

当前的第一轮模型池为：

- XGBoost
- RF
- CatBoost
- LightGBM
- BP

### 6.3 后续实验组织方式已经确定

整体将按如下顺序展开：

1. 继续完成 `plan/next_stage_ml_optimization_plan.md` 中的关键实验
